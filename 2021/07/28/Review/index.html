who，why，how

1. 复习Mybatis

1、什么是mybatis

半自动化的ORM持久层框架 底层是对JDBC的封装

2、mybatis的优缺点

缺点:

​ 移植性差(SQL语句依赖于数据库,不能随意更换数据库)

​ 工作量大(SQL语句编写需要厉害的人)

优点:

​ sql与程序分离，sq|可以重复使用，降低耦合

​ 支持动态sql

3、Mybatis 与hibernate有哪些不同

Hibernate学习成本很大，有自己的hql语句，框架笨重

Hibernate:从sql生成，解析，执行都是Hibernate自己做

Mybatis:sql是自己编写，解析和执行是Mybatis自己完成

传统的JDBC是手动的搓衣板

Hibernate是全自动化的

Mybatis是半自动化的

4、通常一个xml映射文件,都会对应一个Dao接口，Dao接口的方法中，参数不同，方法能重载吗?

不能重载，因为xml中的id是唯一值

5、mybatis是如何将 sql 执行结果封装为目标对象并返回的?

resultType、resultMap

6、mybatis的懒加载， 延迟加载?

什么时候用，根据用户的需求看那些查询需要提高效率，分步查询的时候才可以用

7、关于缓存

一级缓存、二级缓存

一级缓存: sqlsession级别的，是一直开启的， 无法关闭

​ 一级缓存失效的情况 (没有用到当前的缓存，又向数据库发请求)

1.不同的sqlsession

2、sqlsession相同， 查询条件不同，查个10，20

3、sqlsession相同， 在两次相同查询中间,执行了一次增删改

4、sqlsesson.clearCache()

==❤== 二级缓存:

namespace级别的 不同的namespace查出来的数据，会放在不同的二级缓存中

机制:

1、一个会话，查询一条数据，这个数据就会被放在当前会话的一级缓存中

2、如果会话关闭，一级缓存中的数据就会被保存到二级缓存中，新的会话查询数据，就可以从二级缓存中获取

2. 复习Spring

1、Spring是什么？

spring是一个开源的轻量级 IOC 和 AOP 的容器框架

容器:管理应用对象的生命周期

spring是个分层结构，以ioc和aop为内核，每一层都提供了解决技术

  • Web层: springMVC

  • 持久层: jdbcTemplet

  • 业务层: ioc，aop，事务管理

1、IOC、 DI

  1. ioc:控制反转

  2. 控制反转:把创建对象和对象之间的维护的权限转交给spring来处理

  3. 传统的:村长自己抱着消毒箱去卫计委要口罩，有口罩了自己把箱子拉回来

  4. ioc:弄个消毒箱放在村长的办公室，卫计委来巡视，看到有箱子，自动放满口罩容器:卫计委，政府给你自动发放口罩
    DI,依赖注入:依赖注入比控制反转更能确切的表达Spring这种模式

  5. ioc怎么在代码中体现: mapper接口， 注入到容器中

2、Aop
  1. aop:面向切面
  2. AOP是面向切面编程，是对OOP(面向对象)的补充，为了使代码的逻辑结构更清晰
  3. 好处:业务逻辑更简洁，只包含核心业务代码
  4. aop的底层原理:

使用动态代理模式在方法执行前后或出现异常的时候，加入相关的逻辑代理模式:静态代理，动态代理

用户 中介 房东

中介:代理

房东:代理的

对象用户: 目标对象’

中介取代了房东，由中介来决定什么时候去看房，去见哪个房东

用户和房东签合同的前后，中介可以操作

3、事务

Spring事务的管理机制:

spring并不会直接管理事务，而是提供了多种事务管理器，事务管理的职责委托给JDBC、Mybatis等持久化机制多提供的相关平台框架的事务来实现。
